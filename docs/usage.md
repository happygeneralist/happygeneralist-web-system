# Usage

The MVP is intended as a reference and starter system.

## Current use pattern

Copy the layouts, components, styles and schemas into a project repo when needed.

The main intended project is `happygeneralist-site`.

## Later use pattern

When the system stabilises, choose whether to turn it into:

- a GitHub template repo
- an npm package
- a private/public Astro integration
- a source-of-truth documentation site

Do not package it too early.

## Suggested adoption flow

1. Use the examples to test the pattern.
2. Copy the useful pieces into a real project.
3. Refactor repeated pieces back into this repo.
4. Document the convention.
5. Package only after the convention has survived real use.
