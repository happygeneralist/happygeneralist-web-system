# Principles

## Stay close to standards

Use GOV.UK Frontend and public-sector accessibility conventions as a foundation.

Do not fork, imitate or dilute GOV.UK branding.

## Design for clarity first

Standard pages should be readable, accessible and content-led.

Use visual expression only when it helps people understand something that text alone cannot carry well.

## Treat semantics as part of the system

This system is not just visual.

It should model content type, maturity, provenance, relationships, use and reader state.

## Keep Labs expressive but disciplined

Labs can be modular, spatial and visually explanatory.

They should not become marketing pages or decorative showcases.

## Prefer boring infrastructure

Use Astro, Markdown/MDX, simple TypeScript and static deployment.

Avoid heavy dependencies unless the need is clear.
